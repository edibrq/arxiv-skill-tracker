## 📄 Item 1: EvoSkill - Self-Evolving Agent Skills

**Source:** 36kr.com (Chinese Tech News)  
**URL:** https://www.36kr.com/p/3728164375231362  
**ID:** 36kr-3728164375231362  
**Authors:** Research team from Sentient & Virginia Tech  
**Published:** 2026-03-21  
**Type:** Article (Research Summary)

### Abstract/Summary

EvoSkill is a self-evolving system that automatically discovers and optimizes Agent Skills through failure analysis. The framework addresses the limitation of manually-written domain-specific skills, which are costly and hard to scale.

**Key Contributions:**
- Three-agent collaborative evolution cycle:
  - **Executor Agent (A):** Handles tasks with existing Skills library, generates execution traces
  - **Proposer Agent (P):** Analyzes failure cases, diagnoses capability gaps, proposes new/modified skills
  - **Skills Builder Agent (S):** Converts abstract proposals into concrete Skills folders with metadata, instructions, and code
- Bottom model remains frozen; only Skills library updates through iteration
- Feedback history tracks each proposal and its outcome to avoid repeating ineffective solutions

**Results:**
- OfficeQA (financial document QA): Accuracy improved from 60.6% → 67.9%
- SealQA (adversarial search QA): Accuracy improved from 26.6% → 38.7% (+12.1 points)
- Zero-shot transfer to BrowseComp: 43.5% → 48.8% (+5.3 points)

**Key Insight:** Skills discovered through EvoSkill are cross-task transferable, demonstrating generalizability beyond original training scenarios.

## 📄 Item 2: Agent Skill Framework for Small Language Models

**Source:** 53ai.com (AI Research News)  
**URL:** https://www.53ai.com/news/LargeLanguageModel/2026022679682.html  
**ID:** 53ai-2026022679682  
**Authors:** Research team from Luxembourg University, Foyer S.A., Princeton University, Paris-Saclay University  
**Published:** 2026-03-21  
**Type:** Article (Paper Discussion)

### Abstract/Summary

This article discusses the paper "Agent Skill Framework: Perspectives on the Potential of Small Language Models in Industrial Environments" exploring whether Small Language Models (SLMs) can benefit from Agent Skill frameworks.

**Key Contributions:**
- First rigorous mathematical definition of Agent Skill process as a **Partially Observable Markov Decision Process (POMDP)** with information acquisition actions
- Proves that progressive disclosure in Agent Skills is optimal control behavior, not just engineering trick
- Systematic evaluation scheme for SLMs including task accuracy, skill selection accuracy, and **GPU VRAM time (GB-min)** metric
- Deep post-hoc exploration on real industrial dataset (InsurBench)

**Three Context Engineering Strategies Compared:**
| Strategy | Core Mechanism |
|----------|----------------|
| **Agent Skill Instruction (ASI)** | "Judge-Retrieve-Generate" 3-stage lightweight flow; +23.6% accuracy, +11% latency |
| Full Skill Instruction (FSI) | All skills disclosed upfront |
| Direct Instruction (DI) | No external skill support |

**Key Findings:**
- Most SLMs show significant performance improvement under ASI strategy
- Qwen3-80B-Instruct on FiNER: 0.198 → 0.654 accuracy
- Very small models (4B, 270M) benefit most from chat history
- Skill keyword substitution ("Skill" → "Expertise", "Capability", "Proficiency") has minimal impact on performance

**Industrial Implications:** Agent Skill frameworks enable SLMs to achieve competitive performance in data-sensitive industrial scenarios (finance, military) where continuous API calls to closed-source models are not feasible.
